Style Guide

guidelines

Adopt-a-Tree uses Sass as a preprocessor for this project's CSS.

Watch and compile the Sass files using the command line, or a GUI application like Prepros, or a task runner like Gulp, or a bundler like Webpack.

This project uses Vite.js

Do use single-line Sass comments (which are the same as single-line JavaScript comments) for comments that other developers need to see, since they won't get compiled into the CSS stylesheet.

Don't edit the generated CSS files.

Don't use inline styles. The handful of inline styles in the style guide are purely for demo.

Do use tab to indent your code.

Do follow hyphen-separated class naming. E.g. list-block.

Modifier classes use a double-hyphen naming convention. E.g.list-block--profile.

Do create a separate class or ID for an element which requires JavaScript to modify its behavior. Prefix it with js- and don't attach any styling to this class or ID.

Do browse through the variables, globals, spacing and typography partials. They contain plenty of reusable code.

Do use camelCase for naming variables and functions in JavaScript.

Do use semicolons after JavaScript statements.

Do strive for an accessible user interface for users of keyboards and screen readers.

Do ensure the project works on modern browers, including IE Edge, Chrome, Firefox, Safari, Android and iOS.

colors

Use variables or classes for colors.

Background colors have a -bg suffix.

To incorporate a new color, add the color variable to the variables partial and refer to it in the Sass map inside the colors partial.
